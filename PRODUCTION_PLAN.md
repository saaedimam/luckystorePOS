## Phase 1: Foundation Fixes (Week 1) — *Ship a usable admin panel*

### 1.1 Style Unification
**Problem:** Three styling systems fighting each other (CSS vars + inline + Tailwind).
**Fix:**
- Pick **one**: CSS custom properties (tokens) → Tailwind integration. Remove all inline `style={{}}` from components.
- Convert CollectionsWorkspace, CustomerLedgerPage, SupplierLedgerPage from dark-theme hardcoded classes to use design tokens
- Delete dead POS components (BillingPanel, ProductCard, CategoryPill) that aren't used
- Add POS-specific CSS module or extract QuickPosPage inline styles

### 1.2 Currency & i18n
- Replace all `$` with `৳` across admin web
- Add a currency formatter utility so it's one place to change
- Date formatting: use consistent `dd/MM/yyyy` for BD locale

### 1.3 Auth & Security
- Move `/pos` inside `<AuthGuard>` — currently accessible without login
- Wire the logout button (`signOut()` from AuthContext)
- Add error boundary component wrapping `<Routes>`
- Add session expiry redirect (Supabase token refresh failure → login)

### 1.4 Dead Code Cleanup
- Delete `apps/admin_web/src/components/pos/BillingPanel.tsx` (unused)
- Delete `apps/admin_web/src/components/pos/ProductCard.tsx` (unused)
- Delete `apps/admin_web/src/components/pos/CategoryPill.tsx` (unused)
- Delete `apps/admin_web/src/components/pos/QuantityStepper.tsx` (unused)
- Delete unused Storybook stories (`*.stories.tsx`)
- Remove stray `data/my.karbarapp.com-Karbar-cvscreenshot (3).jpeg`
