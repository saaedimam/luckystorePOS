# Lucky Store POS - Pilot Program Criteria

**Purpose:** Define selection criteria and expectations for the initial 10 pilot stores.

**Program Duration:** 30 days (Day 1 - Day 30)

**Program Start Date:** [TO BE DETERMINED]

---

## 1. Pilot Program Overview

### Objective
Validate Lucky Store POS in real-world retail environments, gather feedback, and refine the system before public launch.

### Timeline
- **Selection:** Days 1-5
- **Onboarding:** Days 6-10
- **Usage Period:** Days 11-40
- **Feedback & Iteration:** Days 41-90
- **Go-Live Preparation:** Day 91+

### Benefits for Pilot Stores
| Benefit | Description |
|---------|-------------|
| **Free Usage** | Complete system access for 30 days, no cost |
| **Priority Support** | Dedicated support channel with 1-hour response time |
| **Custom Training** | On-site training for staff (half-day session) |
| **Feedback Rewards** | Referral bonus for successful recommendations |
| **Early Adoption Discount** | 50% off first year subscription if converted |

## 2. Selection Criteria

### Store Profile Requirements

#### Type (Choose Any One)
- [x] **Corner Store / Mini Mart** - 500-1,000 SKUs, 1-2 POS terminals
- [x] **Electronics Retailer** - 1,000-2,000 SKUs, 2-3 POS terminals
- [x] **Clothing Boutique** - 500-1,500 SKUs, 1-2 POS terminals
- [x] **Grocery Store** - 1,000-3,000 SKUs, 2-4 POS terminals
- [x] **Convenience Store** - 300-800 SKUs, 1 POS terminal

#### Location (Within City Limits)
- [x] Primary service area: Dhaka (or specified city)
- [x] Internet connectivity: At least 3G/4G available
- [x ] Accessibility: Staff available for training sessions

#### Current Challenges (Must Have At Least 2)
- [x] Frequent internet downtime affecting sales
- [x] Inventory accuracy issues (overselling, shrinkage)
- [x] Lack of real-time sales visibility
- [x] Manual due payment tracking
- [x] Staff training on existing POS is difficult

#### Commitment Level
- [x] Willing to use exclusively for 30 days (no backup POS)
- [x] Available for weekly check-in calls (15-20 minutes)
- [x] Will provide detailed feedback (via form + call)
- [x] Allow us to document usage (with permission)
- [x] Contactable during training days

## 3. Application Process

### Step 1: Initial Application
**Form:** Online or in-person

**Required Information:**
```
1. Store Details
   - Store Name
   - Owner/Manager Name
   - Contact Number
   - Email Address
   - Full Address

2. Current Setup
   - Number of POS terminals used (if any)
   - Number of cashiers
   - Average daily transactions
   - Inventory size (approx. SKUs)

3. Pain Points
   - Top 3 challenges with current POS
   - Frequency of internet issues
   - Inventory accuracy concerns
   - Sales monitoring needs
```

### Step 2: Interview (20-30 minutes)
**Conducted by:** Sales/Implementation team

**Discussion Points:**
- Current workflow and pain points
- Tech comfort level of staff
- Expected use cases
- Timeline expectations

### Step 3: Selection
**Selection Committee:** Founder + Product Manager + Implementation Lead

**Decision Criteria:**
- Alignment with target profile (70%)
- Commitment level (20%)
- Diversity of store types (10%)

**Notification Timeline:** Within 48 hours of interview

## 4. Onboarding Process

### Day 1-2: Account Setup
- [ ] Create store account
- [ ] Configure store settings
- [ ] Import inventory (CSV or manual)
- [ ] Set up user accounts for cashiers/managers
- [ ] Configure printer settings

### Day 3: Hardware Setup
- [ ] Install app on devices
- [ ] Connect barcode scanner
- [ ] Connect receipt printer
- [ ] Configure network settings
- [ ] Test all components

### Day 4: Staff Training (3 hours)
**Training Modules:**
1. **Basics (1 hour)**
   - Turning on/off the system
   - Scanning items
   - Processing sales
   - Cash handling

2. **Advanced (1 hour)**
   - Discount application
   - Payment modes
   - Returns/exchanges
   - Closing review

3. **Hands-On Practice (1 hour)**
   - Mock sales with all scenarios
   - Offline mode simulation
   - Troubleshooting basics

**Training Materials:**
- User manual (PDF)
- Video tutorials (3-5 minute each)
- Quick reference card (printed)

### Day 5-6: Go-Live Support
- [x] On-site support availability
- [x] Real-time troubleshooting
- [x] First 50 transactions monitored
- [x] Daily check-in with owner

## 5. Feedback Collection

### Weekly Feedback Forms

#### Week 1 (Days 7-13)
**Focus:** Initial onboarding experience

**Questions:**
1. How smooth was the setup process? (1-5)
2. What was the hardest part to understand?
3. What features are you using most?
4. Any bugs or errors encountered?
5. Staff adoption rate? (1-5)

**Submission Method:** In-app form + email

---

#### Week 2-3 (Days 14-27)
**Focus:** Core functionality

**Questions:**
1. How many successful transactions?
2. Offline mode usage (how many times)?
3. Inventory accuracy improvements?
4. Daily report usefulness? (1-5)
5. Due reminder effectiveness? (1-5)
6. Bar code scan success rate? (%)

**Submission Method:** Weekly check-in call + form

---

#### Week 4 (Days 28-30)
**Focus:** Overall satisfaction & conversion

**Questions:**
1. Would you continue using after pilot? (Yes/No/Maybe)
2. Likelihood to recommend to others? (0-10)
3. Top 3 benefits you experienced
4. Top 3 areas for improvement
5. Would you pay XTK/month for this system?
6. Any dealbreakers?

**Submission Method:** In-person interview + form

---

## 7. Post-Pilot Transition

### Option A: Full Conversion
**Timeline:** Days 31-45

**Actions:**
1. **Subscription Setup**
   - Sign service agreement
   - Select pricing plan (pilot discount applies)
   - Set up billing

2. **System Hardening**
   - Implement pilot feedback
   - Fix critical bugs
   - Add requested features (if feasible)

3. **Ongoing Support**
   - Standard support tier
   - Monthly check-in calls
   - Quarterly feature reviews

---

### Option B: Extended Pilot
**Timeline:** Days 31-60

Eligible if:
- Minor bugs preventing full adoption
- Integration complexities
- Additional training needed

**Requirements:**
- Written extension agreement
- Continued feedback submissions
- Weekly progress reports

---

### Option C: Withdrawal
**Timeline:** Days 31-35

If store chooses not to continue:
- Graceful decommissioning
- Data export provision
- Feedback debrief session

## 8. Pilot Store Referral Program

### Incentive Structure

**For Pilot Store:**
- **Referral Bonus:** 5TK discount/month for each successful referral
- **Maximum Bonus:** 10TK/month (for 2 referrals)
- **Duration:** Bonus applies for 12 months

**For Referred Store:**
- **Referral Discount:** 25TK discount in first 2 months
- **Exclusivity:** Bonus applies only if referred store is in different location

### Referral Process
1. Pilot store submits referral via in-app portal
2. Referred store completes application
3. Upon conversion, bonus credited to pilot store account
4. Referral discount applied to new store
