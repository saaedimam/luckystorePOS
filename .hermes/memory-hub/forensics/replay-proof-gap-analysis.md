# Replay Proof Gap Analysis

Authoritative taxonomy of established evidence vs unproven presumptions following the repair cycle.

## 1. THE KNOWN (FACT)
- **Fact 1**: Queue iteration ALWAYS executes in `sequenceId` ascending order via `_sortQueueBySequenceId()`.
- **Fact 2**: Serialization enforces 15-retry hard cap before auto-conversion to `unknownFailure` terminal state.
- **Fact 3**: Expiring lease algorithm calculates `DateTime.now() + 5m` and auto-reclaims `syncing` items on the NEXT tick of `_syncQueue()`.
- **Fact 4**: The eval script successfully loads TS runtime dependencies (`crypto`, `path`, `dotenv`) and is syntactically restored.
- **Fact 5**: Mobile transaction sync executes EXCLUSIVELY on the main UI isolate, making it collision-free on a single device runtime.

## 2. THE ASSUMED (INFERENCE)
- **Inference 1**: It is assumed that Dart's `SocketException` and `HttpException` map fully to user's localized network failure scenarios.
- **Inference 2**: It is assumed that the local device clock used for `leaseExpiresAt` derivation matches local time consistency across app reboots.
- **Inference 3**: It is assumed that re-enabling `testSerializationCollision()` would yield success against the active Supabase Staging target.

## 3. THE MISSING (UNKNOWN)
- **Unknown 1**: Handling of duplicate `sequenceId` generated by collision of `createdAt.millisecondsSinceEpoch` across pre-R1 legacy queue items.
- **Unknown 2**: Dynamic performance of local `_persistQueue()` JSON-write cycle under heavy multi-hundred sale queues.
- **Unknown 3**: True operational resilience to sudden connection severance during midpoint of `rpc('complete_sale')`.
